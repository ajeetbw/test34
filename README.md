# Sprint Call Data Record Processor
## Project Overview
The Sprint Call Data Record (CDR) Processor is a comprehensive solution designed to handle CDR files that contain charging event records for cellular service providers. The primary function of this project is to process CDR files, which are generated by the network and contain descriptions of charging events such as voice calls and SMS messages with all relevant data. The processed data is then used for customer billing and inter-operator settlement.

## Project Description
The Sprint CDR Processor is built to manage offline charging, a process where charging information for network resource usage is collected concurrently with that resource usage. The charging information is then passed through a chain of logical charging functions, resulting in the generation of CDR files. These files are then transferred to the network operator's billing domain for the purpose of subscriber billing and inter-operator accounting.

### CDR File Format
The CDR files contain a header section followed by a variable-sized CDR data section. The CDR data section contains zero or more concatenated CDR records, each encoded on a single line. A CDR record is composed of the following fields, separated by `|`:
* MSISDN: unique number identifying a subscription in a mobile network (maximum length of 7 digits)
* Operator Brand Name: a string (maximum 64 characters)
* Operator MMC/MNC: mobile country code and mobile network code (maximum 6 digits)
* Call type: one of the following:
	+ MOC: outgoing voice call
	+ MTC: incoming voice call
	+ SMS-MO: outgoing message
	+ SMS-MT: incoming message
	+ GPRS: internet
* Duration: in seconds
* Download: MB downloaded (if type is GPRS)
* Upload: MB uploaded (if type is GPRS)
* Third party MSISDN: corresponding third party in this call (empty for GPRS)
* Third party operator MMC/MNC: corresponding third party's mobile operator in this call

### Tech Stack
The Sprint CDR Processor is built using the following technologies:
* **Programming Languages:** Java, Python
* **Frameworks:** Spring Boot, Apache Spark
* **Libraries:** Apache Commons, Jackson

## Installation and Startup
To install and start the Sprint CDR Processor, follow these steps:
1. Clone the repository using `git clone`
2. Install the required dependencies using `mvn install` (for Java) or `pip install -r requirements.txt` (for Python)
3. Configure the application properties (e.g., database connection, file paths)
4. Start the application using `java -jar target/sprint-cdr-processor.jar` (for Java) or `python app.py` (for Python)

## Usage
The Sprint CDR Processor provides the following functions:
1. Customer Billing: processes CDR records to generate billing data for customers
2. Inter Operator Settlement: processes CDR records to generate settlement data for inter-operator accounting

### API Examples
The following API examples demonstrate how to use the Sprint CDR Processor:
* **Customer Billing:** `POST /billing` with a JSON payload containing the CDR record
* **Inter Operator Settlement:** `POST /settlement` with a JSON payload containing the CDR record
